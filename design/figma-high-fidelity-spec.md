# Figma 高忠実度モック仕様（初稿）

## 目的
- ユーザーテストと最初のレビュー用に高忠実度デザインを用意する。
- 実装チームに渡せるコンポーネント設計とアニメーション仕様を含める。

## 画面：ランディング（高忠実度）
- 画面幅：最大1200px、モバイル幅375px のレスポンシブ設計
- ヒーロー：SVGロゴ（左）／CTA（開始ボタン：グラデーション + マイクロアニメ）
- 視覚要素：Lottie の軽いモーション（キービジュアル）
- テキストスタイル：H1 40px / H2 24px / body 16px

## 画面：入力フォーム
- 学籍番号と氏名を大きめのフォームフィールドで配置（ラベルは上置き）
- フィールドエラー時は赤色のアウトラインと補助テキストを表示
- 同意チェックは小さな情報アイコンで説明を追加

## 画面：質問フロー
- 1問ずつカード表示、背景はぼかしセクション
- プログレスバーは上部固定、進行時はアニメで幅を増す
- 選択肢はカード形式（アイコン + テキスト）、選択時にスケールアップと影を追加
- スワイプで遷移（モバイル）／矢印アニメ（デスクトップ）

## 画面：結果ページ
- Lottieアニメ（成功モーション）を大きく表示
- 主要役割は大きなカードで上位3つを表示（％はカウントアップアニメ）
- 各カードに「行動提案」ボタン（クリックで詳細モーダル）
- 共有ボタン（リンク生成と管理者のCSVエクスポート）

## 管理画面（デザイン）
- テーブル＋フィルタUI（期間・役割・学籍番号）
- 詳細モーダル：回答一覧、スコア内訳、ダウンロードリンク
- ログビュー：閲覧履歴・エクスポート履歴を確認可能

## コンポーネント（Figmaで作成）
- Buttons（Primary / Secondary / Ghost / Disabled）
- Form Inputs（Normal / Error / Focus）
- Card（Question / Result / Role）
- Modal（Large / Small）
- Table（Header / Row / Empty State）
- Icons（Role icons, Info）

## アニメーション詳細（Figmaから実装へ）
- 質問遷移：x軸スライド（distance 24px, duration 240ms, easing cubic-bezier(0.22, 1, 0.36, 1))
- 選択アクション：scale 1 → 1.02, box-shadow opacity 0→1, duration 120ms
- 結果％カウント：0→N, duration 800ms, ease-out

## アクセシビリティ
- フォントサイズとカラーコントラストはWCAG AA基準を満たす
- キーボードフォーカスのスタイルを明示（outline + shadow）

## 共有方法
- Figmaファイルを作成してレビューリンクを発行（ステークホルダーへ共有）
- 初回はPNGスクリーンショットをここに添付してレビューしてもらいます
